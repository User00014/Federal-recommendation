## 2. 现在的模型架构（推荐模型）

模型文件：`src/models.py`

### 2.1 结构分解

1. 协同过滤部分：
- 用户嵌入：`user_emb`
- 物品嵌入：`item_emb`

2. 内容特征部分：
- 侧信息编码：`feat_encoder`
- 输入特征来自 `genre + tag + year` 的拼接向量

3. 共享主干（可聚合）：
- `shared_backbone`
- `shared_head`

4. 本地个性化头（不聚合）：
- `personal_head`
- 仅在客户端本地更新和保存

### 2.2 前向公式（简化）

设第 \(u\) 个用户与第 \(i\) 个物品：
$$
\mathbf{e}_u = Emb_u(u), \quad \mathbf{e}_i = Emb_i(i), \quad \mathbf{f}_i = W_f x_i
$$
\[
$$
\mathbf{h} = Backbone([\mathbf{e}_u, \mathbf{e}_i, \mathbf{f}_i])
$$
\]

\[
$$
\hat r_{ui} = Head_{shared}(\mathbf{h}) + \mathbb{1}_{\text{personal}} \cdot Head_{local}(\mathbf{h})
$$
\]

其中 \(\mathbb{1}_{\text{personal}}\) 表示是否开启个性化头。

## 3. 联邦学习框架（当前实现）

核心文件：`src/server_client.py`、`main.py`

### 3.1 参与角色

- Server：维护全局共享参数，执行聚合
- Client：本地训练并上传共享参数更新
- Attacker：基于梯度特征进行成员推断攻击

### 3.2 一轮训练流程

1. 服务器广播全局模型 \(w_t\)
2. 采样客户端集合 \(\mathcal{S}_t\)
3. 每个客户端本地训练，得到共享参数更新 \(\Delta_k\)
4. 服务器聚合：
    \[
$$
  \bar\Delta_t = \frac{1}{|\mathcal{S}_t|}\sum_{k\in \mathcal{S}_t}\Delta_k
$$
  \]
5. 服务器更新：
    \[
$$
  w_{t+1} = w_t + \bar\Delta_t
$$
  \]
6. 记录指标（RMSE、ASR、AUC 等）

### 3.3 FedProx 原理（非IID稳态）

若开启 `FL_ALGO=FEDPROX`，客户端优化目标为：

\[
$$
\min_w\; F_k(w) + \frac{\mu}{2}\|w - w_t\|_2^2
$$
\]

- \(F_k(w)\)：客户端本地损失
- \(w_t\)：当前全局参数
- \(\mu\)：近端项强度（`PROX_MU`）

作用：约束本地模型偏离全局模型过远，减轻 non-IID 导致的漂移。

## 4. 隐私保护机制与数学原理

核心文件：`src/privacy.py`

### 4.1 梯度裁剪（Clipping）

对更新 \(g\) 进行 L2 裁剪：

\[
g_{clip} = \frac{g}{\max(1, \|g\|_2/C)}
\]

其中 \(C\) 是 `CLIP_NORM`。

### 4.2 高斯加噪（Gaussian Mechanism）

\[
\tilde g = g_{clip} + \mathcal{N}(0, \sigma^2 I)
\]

其中 \(\sigma\) 对应 `DP_SIGMA` 或自适应后的 \(\sigma^*\)。

### 4.3 自适应噪声（当前实现）

代码中对每层采用动态噪声：

\[
$$
\sigma^* = \text{clip}_{[\sigma_{min},\sigma_{max}]}
\Big(
\sigma_0 \cdot f_{progress} \cdot f_{sens} \cdot f_{sparse}
\Big)
$$
\]

其中：

- 进度因子（前强后弱）：
  \[
  $$
  f_{progress} = 1 + \lambda(1-p),\quad p=\frac{t}{T-1}
  $$
  \]
- 敏感度因子（近似）：
  \[
  $$
  f_{sens} = \text{clamp}\left(\sqrt{\|g\|_2/C}, 0.8, 1.5\right)
  $$
  \]
- 稀疏增强因子：当非零比例很低时，额外增噪

### 4.4 CDP vs LDP

- `CDP`：聚合后加噪
  - 优点：效用通常更好
  - 假设：服务器可信
- `LDP`：客户端上传前加噪
  - 优点：保护边界更强（零信任）
  - 缺点：噪声累积，精度下降更明显

## 5. 攻击模型与评估

核心文件：`src/attack.py`

### 5.1 攻击思路

成员推断攻击（MIA）：判断某用户样本是否参与训练。

攻击输入不是原始梯度，而是“稀疏感知统计特征”：
- 非零比例
- 活跃梯度范数 / 均值 / 标准差 / 分位数
- 分布极值

### 5.2 评估指标

- ASR（Attack Success Rate）
\[
ASR = \frac{\#\text{correct predictions}}{\#\text{all predictions}}
\]
越接近 0.5 越安全。

- AUC：分类可分性，越接近 0.5 越安全。
- Precision / Recall / F1：补充攻击行为刻画。

## 7. 技术创新点（论文可直接写）

1. **FedProx 稳态优化接入联邦推荐**：缓解 non-IID 客户端漂移。
2. **共享参数与个性化参数解耦**：本地个性化头不参与聚合，提升 DP 条件下的效用。
3. **自适应差分隐私机制**：将固定噪声升级为与训练阶段、敏感度、稀疏性耦合的动态噪声。
4. **攻防评估体系扩展**：从单一 ASR 扩展至 ASR/AUC/F1 与系统开销联合评估。
5. **论文级可视化体系**：从“单曲线”升级到“创新性可解释图谱”。

## 8. 关键参数含义与调参建议

| 参数 | 含义 | 建议范围 | 风险提示 |
|---|---|---|---|
| `ROUNDS` | 通信轮数 | 200 / 500 | 太小会不稳定，太大会耗时 |
| `USERS_PER_ROUND` | 每轮客户端数 | 20~40 | 太小波动大 |
| `LOCAL_EPOCHS` | 本地训练轮数 | 3~8 | 太大易客户端漂移 |
| `LR` | 学习率 | 1e-4 ~ 5e-4 | 太大可能震荡 |
| `FL_ALGO` | `FEDAVG/FEDPROX` | 优先 `FEDPROX` | `FEDAVG` 在 non-IID 下更脆弱 |
| `PROX_MU` | 近端约束强度 | 0.001~0.05 | 太大可能欠拟合 |
| `ENABLE_PERSONALIZATION` | 个性化头开关 | 建议 ON | 关闭后DP下精度常更差 |
| `PRIVACY_MODE` | `PLAIN/CDP/LDP` | 分组实验切换 | 注意信任假设差异 |
| `DP_SIGMA` | 基础噪声强度 | CDP:0.002~0.01 / LDP:0.02~0.05 | 太大易崩 |
| `CLIP_NORM` | 裁剪阈值 | 0.003~0.02 | 太小信息损失大 |
| `ENABLE_ADAPTIVE_DP` | 自适应DP开关 | 核心实验 ON | 需与固定噪声组对照 |
| `DP_SIGMA_MIN/MAX` | 动态噪声边界 | 0.001 / 0.1 | 防止过度调节 |
| `DP_PROGRESSIVE_DECAY` | 前强后弱幅度 | 0.2~0.6 | 过高可能后期保护不足 |
| `DP_SPARSITY_BOOST` | 稀疏增强 | 0.05~0.25 | 过高会伤害长尾效用 |

## 9. 实验路径（推荐执行顺序）

> 建议先跑 12 组，后续再做 500 轮复现实验。

### 9.1 冒烟组（2组）

- S1：`PLAIN + FEDAVG + NP + FDP`（20轮）
- S2：`CDP sigma=0.005 + FEDPROX + P + ADP`（20轮）

目的：确认训练、保存、绘图全流程无误。

### 9.2 主对比组（8组，200轮）

- G0：`PLAIN + FEDAVG + NP + FDP`✅️
- G1：`PLAIN + FEDPROX + NP + FDP`✅️
- G2：`PLAIN + FEDPROX + P + FDP`✅️
- G3：`CDP sigma=0.002 + FEDPROX + P + FDP`✅️
- G4：`CDP sigma=0.005 + FEDPROX + P + FDP`✅️
- G5：`CDP sigma=0.01 + FEDPROX + P + FDP`✅️
- G6：`CDP sigma=0.005 + FEDPROX + P + ADP`  ← 核心创新组✅️
- G7：`LDP sigma=0.02 + FEDPROX + P + FDP`✅️
- G8：`LDP sigma=0.05 + FEDPROX + P + FDP`
- G9：`LDP sigma=0.02 + FEDPROX + P + ADP`

### 9.3 消融组（4组，200轮）

- A1：在 G6 上关闭个性化头（`P -> NP`）
- A2：在 G6 上换优化器（`FEDPROX -> FEDAVG`）
- A3：在 G6 上关闭自适应DP（`ADP -> FDP`）
- A4：在 G6 上改 `PROX_MU`（0.01 -> 0.001 或 0.05）

### 9.4 定稿复现实验（3组 × 3seed，500轮）

- 组：`G0 / G6 / G9`
- seed：`42 / 52 / 62`

## 10. 结果判定标准（什么算合理）

1. **PLAIN组**：
- ASR 通常显著高于 0.5（常见 0.65~0.85）
- RMSE 作为效用上界

2. **CDP组**（尤其 0.002~0.01）：
- ASR 应明显向 0.5 靠近
- RMSE 相比 PLAIN 可接受上升
- 一般是“隐私-效用最平衡”候选

3. **LDP组**：
- ASR 可下降，但常不如 CDP 接近 0.5
- RMSE 通常劣于 CDP
- 高噪声下可能出现不收敛或严重退化（合理现象）

4. **创新模块预期**：
- `G1` 优于 `G0`（更稳）
- `G2` 优于 `G1`（个性化提升效用）
- `G6` 在隐私接近时优于 `G4`（ADP有效）

## 12. 当前实现边界（写论文时要说明）

1. 代码使用高斯噪声机制，但尚未集成完整的 `(ε,δ)` 隐私会计器（RDP accountant）；当前以 `sigma` 为控制量进行对比实验。
2. 攻击以成员推断为主，尚未系统纳入梯度重构等更强攻击。
3. 数据为 MovieLens 仿真场景，结论具备方法论意义，但工业落地需重新做参数校准。
