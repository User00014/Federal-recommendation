# 云端批量实验与结果回传说明

## 1. 新增脚本

- `run_cloud_batch.py`：云端连续批量实验（自动串行跑多组，支持断点续跑）
- `pull_cloud_results.ps1`：本地一键拉取云端结果（自动创建新文件夹）
- `src/experiment_io.py`：统一日志命名工具（避免覆盖）

## 2. 日志命名规范

保存名现在包含关键参数，默认格式：

`res_{MODE}_sigma{SIGMA}_{ALGO}_{P/NP}_{ADP/FDP}_{ROUNDS}rounds_seed{SEED}_mu{MU}_{TIMESTAMP}.json`

例如：

`res_CDP_sigma0.005_FEDPROX_P_ADP_1000rounds_seed42_mu0.01_R003_G6_seed42_20260310-210101.json`

## 3. 云端运行命令（Linux）

### 3.1 15组单seed（默认组）

```bash
python run_cloud_batch.py --rounds 1000 --seeds 42 --batch-name batch_15_seed42
```

### 3.2 15组双seed（共30次）

```bash
python run_cloud_batch.py --rounds 1000 --seeds 42,52 --batch-name batch_15_seed42_52
```

### 3.3 后台持续运行（推荐）

```bash
nohup python run_cloud_batch.py --rounds 1000 --seeds 42,52 --batch-name batch_night > batch_night.out 2>&1 &
```

### 3.4 断点续跑

```bash
python run_cloud_batch.py --resume-dir logs/batch_night --retry-failed
```

## 4. 本地回传命令（Windows PowerShell）

在项目目录执行：

```powershell
.\pull_cloud_results.ps1 -Host <云主机IP> -User <用户名> -RemotePath </remote/project/logs/batch_night>
```

脚本会在本地自动创建：

`cloud_results/batch_yyyyMMdd-HHmmss/`

并把整批结果拉到该目录下。

## 5. 绘图兼容性

`visualize.py` 已兼容新命名（含 `seed/mu/timestamp` 扩展字段），可以继续直接从 `logs` 读取绘图。
