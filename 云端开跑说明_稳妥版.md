# 云端开跑（稳妥版）

## 目标

- SSH 上云后，尽快一条链路跑起来
- 自动处理 Python 包和环境
- 避免误在 CPU 上长时间运行
- 支持先跑 1 组、回传、再估时、再跑全量

## 文件

- `requirements_cloud.txt`：云端依赖清单
- `cloud_start_batch.sh`：一键安装 + GPU自检 + 冒烟测试 + 后台启动
- `pull_cloud_results.ps1`：本地拉取（支持持续同步）
- `estimate_runtime.py`：根据已完成组估算总时长/总费用

## 1) 云端首次启动（建议）

```bash
chmod +x cloud_start_batch.sh
PROJECT_DIR=$(pwd) ROUNDS=1000 SEEDS=42 GROUPS=G6 BATCH_NAME=pilot_g6_seed42 RUN_SMOKE=1 ./cloud_start_batch.sh
```

说明：
- `RUN_SMOKE=1` 会先跑 1 轮快速自检，确保包和数据没问题。
- 脚本会检查 CUDA；若不可用会直接退出，避免你花钱跑到 CPU 上。
- 真实实验会在后台 `nohup` 运行，输出在 `logs/nohup_<batch>.out`。

## 2) 查看云端进度

```bash
tail -f logs/nohup_pilot_g6_seed42.out
```

## 3) 本地持续回传（跑一组传一组）

在本地 PowerShell：

```powershell
.\pull_cloud_results.ps1 -Host <云IP> -User <用户名> -RemotePath </云端项目路径/logs/pilot_g6_seed42> -Watch -IntervalSec 300
```

说明：
- 会每 5 分钟拉一次 `batch_state.json` + `batch_summary.csv` + `logs/`
- 检测到所有任务结束后自动停止

## 4) 用首组结果重新估算

本地拿到 `batch_summary.csv` 后：

```bash
py -3 estimate_runtime.py --summary "<本地路径>/batch_summary.csv" --planned-runs 15 --price-per-hour 8.5794
```

如果你后续要跑 30 次（双seed）：

```bash
py -3 estimate_runtime.py --summary "<本地路径>/batch_summary.csv" --planned-runs 30 --price-per-hour 8.5794
```

## 5) 再启动全量批次

```bash
PROJECT_DIR=$(pwd) ROUNDS=1000 SEEDS=42 GROUPS=G0,G1,G2,G3,G4,G5,G6,G7,G8,G9,A1,A2,A3,A4L,A4H BATCH_NAME=full_15_seed42 RUN_SMOKE=0 ./cloud_start_batch.sh
```

## 6) 断点续跑

```bash
source .venv/bin/activate
python run_cloud_batch.py --resume-dir logs/full_15_seed42 --retry-failed
```
